**Status:** ✅ SHIPPED 2026-05-08

## Overview

Walking skeleton for Dog Tinder — all three backend API routes implemented and tested. Proves that a random dog image can be fetched from random.dog, a swipe record can be persisted to disk, and history can be read back, end to end, with no database. Foundation for Phases 2-3 (Login + Swipe UI).

## Phases

### Phase 1: API Layer

**Goal**: The backend API endpoints exist and work — dog images can be fetched and swipes can be saved and read
**Depends on**: Nothing (first phase)
**Requirements**: DOG-01, DOG-02, SWIPE-01, SWIPE-02
**Plans**: 1 plan

Plans:

- [x] 01-01-PLAN.md — Scaffold Next.js + lib/storage.ts + all three API routes (GET /api/dog, POST /api/swipe, GET /api/history)

**Success Criteria (all met):**
1. GET /api/dog returns a JSON response with a valid image URL (jpg/png) and extracted dogId ✓
2. GET /api/dog never returns a .mp4 URL — retries up to 5 attempts ✓
3. POST /api/swipe with a valid body writes a record to /data/swipes.json and returns success ✓
4. GET /api/history returns the full list of swipe records from /data/swipes.json ✓

**Details:**
- Next.js 16 App Router, TypeScript, Tailwind CSS, Vitest
- lib/storage.ts: shared bootstrap-aware JSON I/O (ensureStore, readSwipes, appendSwipe)
- GET /api/dog: random.dog fetch with 5-attempt image-extension retry
- POST /api/swipe: validates all fields, generates server-side timestamp, returns full record
- GET /api/history: reads all records, returns [] on fresh install
- 15 unit/integration tests, all passing
- TDD: RED→GREEN commits for all 3 modules

## Milestone Summary

**Key Decisions:**
- App Router (not Pages Router) — workshop uses latest Next.js patterns
- JSON file storage at /data/swipes.json — no DB overhead for prototype
- sessionStorage for username — clears on browser close, simple
- Buttons for swipe (not drag) — simpler to build
- Skip .mp4 URLs silently — re-fetch is cleaner UX
- data/swipes.json in .gitignore — avoid team conflicts
- GET /api/history separate from POST /api/swipe — single-purpose routes (D-01)
- POST /api/swipe generates timestamp server-side (D-05)
- 5 total retry attempts for /api/dog (D-03)
