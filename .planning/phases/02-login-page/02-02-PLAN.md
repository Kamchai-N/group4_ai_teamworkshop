  <action>
Create two new files. Both follow the same pattern: "use client" + useEffect reads sessionStorage after hydration + router.replace for redirect.

Why useState initial value must be null (not false or ""): null means "not checked yet". The component returns null (renders nothing) until the check runs. This prevents a flash of unprotected content — React renders the component, then runs effects. If the initial state were "" or false, the swipe content might briefly render before the redirect fires.

Why useEffect (not top-level code): sessionStorage does not exist on the server. Calling sessionStorage at module scope or component function body throws ReferenceError during SSR. useEffect only runs in the browser after hydration.

**File 1: app/swipe/page.tsx**

Phase 3 will replace the placeholder div body with the full swipe UI. The guard (useEffect + useState<string | null>) must remain unchanged in Phase 3.

```tsx
"use client";

import { useEffect, useState } from "react";
import { useRouter } from "next/navigation";

export default function SwipePage() {
  const router = useRouter();
  const [username, setUsername] = useState<string | null>(null);

  useEffect(() => {
    const stored = sessionStorage.getItem("username");
    if (!stored) {
      router.replace("/login");
    } else {
      setUsername(stored);
    }
  }, [router]);

  if (username === null) return null;

  return <div>Swipe page — Phase 3</div>;
}
```
